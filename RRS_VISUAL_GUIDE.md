### View Audit Log (SQL)
```sql
-- Connect to database
sqlite3 rrs_enterprise.db

-- View recent actions
SELECT timestamp, username, action, entity_type, entity_id, details
FROM audit_log
ORDER BY timestamp DESC
LIMIT 20;

-- View failed logins
SELECT username, COUNT(*) as attempts
FROM audit_log
WHERE action = 'LOGIN' AND details LIKE '%failed%'
  AND timestamp > datetime('now', '-1 hour')
GROUP BY username;
```

## Common Tasks

### Task: Create New User
```
1. Login as Risk Manager
2. Navigate to "User Management" in sidebar
3. Fill form:
   - Username (unique)
   - Password (strong passphrase)
   - Full Name
   - Email
   - Role (Manager/Contributor/Viewer)
   - Department
4. Click "Create User"
5. Verify success message
```

### Task: Add New Risk
```
1. Login as Risk Manager or Department Contributor
2. Navigate to "Add/Edit Risk" or "Submit New Risk"
3. Fill required fields (*):
   - Risk Name
   - Category
   - Risk Owner
   - Owner Department
   - Likelihood (1-5)
   - Impact Assessment (4 dimensions)
   - Description
4. Optionally add:
   - Causes
   - Triggers
5. Click "Save Risk"
6. Verify risk appears in Risk Register
```

### Task: Review Risks by Department
```
1. Login as any role
2. Navigate to "Risk Register"
3. Use Department filter: Select department
4. Click Apply (automatic filtering)
5. Expand risks (â–¶) to view details
```

### Task: Monitor Critical Risks
```
1. Login as any role
2. Navigate to "Risk Dashboard"
3. Check "Critical Risks" metric card
4. View Risk Heat Map (top-right cells are critical)
5. Review "Top 10 Risks" table (sorted by residual score)
6. Expand risks for details
```

### Task: Check User Activity
```
1. Login as Risk Manager
2. Navigate to "Audit Log"
3. Filter by entity type if needed (e.g., RISK)
4. Review timestamp, user, action, details
5. Look for anomalies (unusual activity, failed logins)
```
